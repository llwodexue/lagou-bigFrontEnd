## 检票名词

![image-20231024171446778](https://gitee.com/lilyn/pic/raw/master/lagoulearn-img/image-20231024171446778.png)

- AFC：自动售检票系统，全称 Automatic Fare Collection，是基于计算机、通信、网络、自动控制等技术，实现轨道交通售票、检票、计费、收费、统计、清分、管理等全过程的自动化系统
- ATMS：自动售票机（Automated Ticketing Machines），通过 ATMs，用户可以选择所需的票种和数量，进行支付，并获得打印好的票据或取票凭证
  - 中间服务层（c程序），负责串行业务，直接调用存储过程处理，某些业务会调用第三方 ctms 售票，比如银通卡业务、E卡通业务等
- CTMS：连接交易管理服务器（Connection and Transaction Management Server）。在实时交易系统中担当交易路由和网关角色，具有交易导航和控制、分布式数据透明访问、负载数据库压力等功能
  - 它通过整合各种功能模块和数据库，提供了全面的售票服务，包括票务销售、座位选择、票价管理、订单处理、支付结算等
  - 售票的c服务（节点转换、节点导航、c程序数据传输）
  
- Psr：持久化存储和检索（Persistence, Storage, and Retrieval）持久化是指将数据从内存中保存到永久性存储介质（如硬盘）以确保数据的可靠性和长期存储
  - Redis 本身提供了持久化选项，可将内存中的数据以不同的方式写入磁盘进行持久化存储，以防止数据丢失。Redis 支持两种主要的持久化方式：RDB（Redis Database）快照和 AOF（Append-Only File）日志
  - 互联网的 redis 集群服务存储售票信息
- 节点：一整套检票系统，包括数据库
- PSAM卡：可编程安全应用模块（Programmable Secure Application Module）卡
  - 它是一种集成了加密算法和安全芯片的智能卡，用于提供安全认证和数据加密等功能
  - 比如：金融IC卡、公交卡、门禁卡（加密的）、学生卡

#### 生成计划

必须先有基本计划

- 列车运行图是指铁路系统中用于规划和安排列车运行的时刻表或计划。它记录了列车在特定时间段内的发车时间、到达时间、途经站点以及预计的行车时间等信息
- 制定列车运行图需要考虑多个因素，包括站点容量、轨道容量、列车速度、交叉口、列车调度策略等

![image-20231024181149811](https://gitee.com/lilyn/pic/raw/master/lagoulearn-img/image-20231024181149811.png)

城际模式有相对时间一说，后台返回的值是相对时间，前端需要转换成实际时间

![image-20231025165427643](https://gitee.com/lilyn/pic/raw/master/lagoulearn-img/image-20231025165427643.png)

相对时间是根据进出站计算出来的，相对的是分钟数（不需要优化成小时分钟）

- 默认值展示：开始时间 05:00，结束时间 23:59（前端展示的时间），不过相对的时间是根据车次信息给的默认值（传给后台的时间）

  - 始发站=当前站，开始时间、结束时间默认2359
  - 终到站=当前站，开始时间、结束时间默认0001
  - 始发站、终到站!=当前站，开始时间默认0001、结束时间默认2359

- 计算规则：

  1. 相对值依照：进站相对值 23:59，出站相对值：00:01

     ```js
     // 1439->2359的分钟数，300->0500的分钟数
     const factInMin = {
       start: (1439 - 300 + '').padStart(4, 0),
       stop: (1439 - 1439 + '').padStart(4, 0)
     }
     const factOutMin = {
       start: (300 - 1 + '').padStart(4, 0),
       stop: (1439 - 1 + '').padStart(4, 0)
     }
     ```

  2. 计算规则：根据相对值计算。进站用相对值减去相对时间，出站用相对值加上相对时间（需要考虑跨天）

#### 动态计划调整

生成之后才可以进行动态计划调整

![image-20231024174900331](https://gitee.com/lilyn/pic/raw/master/lagoulearn-img/image-20231024174900331.png)

- **立刻开检**：选中一条未开检车次，点击“立刻开检”按钮，出现确认提示框，点击“是”，即该选中车次绿色突出显示
- **开检取消**：选择一条已开检车次，点击开检取消，该车次变成未开检状态
- **立刻停检**：选择一条正在开检车次，点击立刻停检，改车次变成已检状态
- **停检取消**：选择一条手工停检车次，点击停检取消，该车次变成开检状态
- **列车晚点**：选择未检车次，点击列车晚点，弹出列车晚点界面。输入实际乘车日期、到达时间、实际发车时间，点击执行按钮，即该车次突出粉红色显示
- **恢复正点**：选择一条晚点车次，点击恢复正点，即该车次恢复到原来正点状态，且粉红色条消失
- **晚点未定**：选择一条车次，点击晚点未定，即该车次变成晚点未定状态，且粉红色突出显示
- **未定取消**：选择一条晚点未定车次，点击未定取消按钮，即该车次变成正点发车状态
- **停运**：选择一条车次，点击停运，进入停运界面。输入起始日期、截止日期，选择周期，点击执行按钮，提示执行成功，该停运车次红色突出显示在主页面的最上侧
- **恢复开行**：选择停运车次，点击恢复开行按钮，进入恢复开行界面。输入起始日期、截止日期，周期，点击执行，该车次即恢复开行
- **检票口调整**：选择某车次，点击检票口调整，进入检票口调整界面。输入起始日期、截止日期，选择检票口，点击执行，该车次检票口调整成功
- **检票时间调整**：选择某车次，点击检票时间调整，进入检票时间调整界面。输入起始日期、截止日期、检票开始时间、检票结束时间，点击执行按钮，即检票时间调整成功
- **改点**：选择某车次，点击改点，即进入改点界面。输入起始日期，截止日期，到达时间，实际发车时间，点击执行，提示执行成功，即改成改点成功
- **计划下载**：选择某车次，点击计划下载，弹出确认对话框，点击“是”，即该车次计划下载到闸机
- **增开临客**：点击增开临客按钮，进入增开临客界面。选择起始日期，截止日期，选择检票类型，选择车次，输入检票开始时间、检票结束时间，选择周期，选择闸机，点击执行按钮，提示成功，即增开临客成功
- **增开折返**：与增开临客同理
- **高级查询**：默认查询范围查不到已检车次

![image-20231024175007904](https://gitee.com/lilyn/pic/raw/master/lagoulearn-img/image-20231024175007904.png)

前端注意点：

- 折返车（planType -> Z），禁止使用 列车晚点、恢复正点、晚点未定、未定取消、列车停运、恢复开行、检票时间调整、改点、同步运行图按钮
- 临客车（planType -> Y），禁止使用 晚点未定、未定取消、同步运行图按钮
- 检票系统执行停运（planType -> N），禁止使用 立即开检、开检取消、立即停检、停检取消、列车晚点、恢复正点、晚点未定、未定取消、检票口调整、检票时间调整、改点、同步运行图
- 客运调令执行停运（planType -> 6），按钮可以全部使用（新改变）
- checkState -> 1、3、4、A，不禁用立即开检按钮
- checkState -> 7，禁用同步运行图按钮

#### SeatType、TicketType

tickeType 与 seatType 处理逻辑类似，seatType 有字母比较特殊，这里以 seatType 举例

- decimalNumber 函数主要是将数组转换为一个长度为 32 的二进制字符串，之后将数组反转，最后再将该二进制字符串转换为十六进制字符串
  - 如果是数字就将这数字作为索引设置为 1，如果是字母就将其转换为 ASCII 编码再减去 55 作为索引设置为 1（A->65）
- echoChecked 反之

```js
export default {
  decimalNumber(arr = []) {
    const valSeq = new Array(32).fill(0)
    for (const k of arr) {
      if (!isNaN(k)) {
        valSeq[k] = '1'
      } else {
        const tempI = k.charCodeAt(0)
        valSeq[tempI - 55] = '1'
      }
    }
    const valStr = valSeq.reverse().join('')
    return parseInt(valStr, 2).toString(16)
  },
  echoChecked(value) {
    const checked = []
    const binSeq = parseInt(value, 16).toString(2)
    if (isNaN(binSeq)) return checked
    const binRev = binSeq.split('').reverse().join('')
    for (let i = 0; i < 63; i++) {
      if (binRev.charAt(i) === '1') {
        if (i < 10) {
          checked.push(i.toString())
        } else {
          const tempI = i + 55
          checked.push(String.fromCharCode(tempI))
        }
      }
    }
    return checked
  }
}
```

## 数据安全

### 加密方式

前端加密方式：

1. 把 **年月日时分秒毫秒** 进行 **BASE64加密** 作为秘钥(secret)

   示例：`20220623093153456 -> MjAyMjA2MjMwOTMxNTM0NTY=`

2. 对原串进行 **DES加密**，模式为 ECB，填充方式为 Pkcs7 生成加密原串(cipher)

   示例：`123 -> wFMwSqOC/QE=`

3. 加密原串(cipher)的 **第1-2位 + 秘钥(secret)截取掉最后一位(截取的是=) + 从第3位开始加密原串(cipher) **作为最终加密串返回

   最终返回示例：`wFMjAyMjA2MjMwOTMxNTM0NTYMwSqOC/QE=`

   ```js
   位数：12345678901234567890123456789012356
   秘钥：20220623093153456 -> 65435139032602202 -> 65435139
   拼接：  MjAyMjA2MjMwOTMxNTM0NTY=          // = 需要去掉拼接到秘串
   原串：wF                       MwSqOC/QE=
   秘串：wFMjAyMjA2MjMwOTMxNTM0NTYMwSqOC/QE=
   ```

### 解密方式

前端解密方式：

1. 把加密串的 **第3-25位截取掉(并加上一个=)作为秘钥**(secret)

   示例：`wFMjAyMjA2MjMwOTMxNTM0NTYMwSqOC/QE= -> MjAyMjA2MjMwOTMxNTM0NTY=`

   ```js
   位数：12345678901234567890123456789012356
   秘串：wFMjAyMjA2MjMwOTMxNTM0NTYMwSqOC/QE=
   拼接：  MjAyMjA2MjMwOTMxNTM0NTY=          // = 需要加上作为拼接串进行解密
   解密：20220623093153456 -> 65435139032602202 -> 65435139
   原串：wF                       MwSqOC/QE=
   ```

2. 把加密串剩余部分（**第1位到第2位的加密串 + 从第25位开始的加密串**）作为加密原串(cipher)

   示例：`wFMjAyMjA2MjMwOTMxNTM0NTYMwSqOC/QE= -> wFMwSqOC/QE=`

3. 对加密原串(cipher)进行 **DES解密**，模式为 ECB，填充方式为 Pkcs7

   最终返回示例：`123`
