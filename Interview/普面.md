## SOP 答题模板

SOP 全称为 Standard Operating Procedure，翻译成中文就是"标准作业程序"。场景题按如下模板回答：

1. **交代背景**：业务场景是什么，遇到了什么问题，影响范围多大
2. **调研方案**：调研了哪些技术方案，各自的优缺点对比
3. **方案落地**：最终选择了什么方案，如何实现的，关键代码/架构
4. **反思总结**：过程中遇到了什么问题，如何解决的，有没有更优方案

## 1000W 行表格如何渲染

**背景**：数据量极大，需要在前端展示和操作千万级数据行，普通 DOM 渲染完全不可用。

**调研方案**：
1. 直接 DOM 渲染：1000W 个 DOM 节点会导致浏览器崩溃，不可行
2. 虚拟表格：只渲染可视区域，滚动时动态替换内容，主流方案
3. Canvas 表格：用 Canvas 绘制，性能远优于 DOM
4. 可视区绘制算法优化：行列虚拟化 + 懒加载
5. Canvas + WebAssembly：skia + WASM 实现极致性能

**方案落地**：
- 封装画布表格引擎，实现行列双虚拟化
- 可视区只渲染约 50 行 × 10 列的单元格
- 使用 Canvas 绘制，配合 requestAnimationFrame 节流
- 支持排序、筛选、合并单元格等交互

**反思**：
- Canvas 无法利用浏览器原生滚动条，需要自定义
- 复杂单元格（如富文本、图表）在 Canvas 中渲染困难
- 后续可考虑 WebAssembly 进一步提升性能

## 超过 Number 最大值怎么处理

JS 安全整数范围是 `Number.MAX_SAFE_INTEGER`（2^53 - 1），超出后精度丢失。

**背景**：
1. 大数据计算场景（金额、ID、科学计算）
2. 后端返回的大数格式化展示
3. 用户输入大数的校验

**解决方案**：
1. `BigInt` 原生支持大整数运算
2. 使用 `decimal.js`、`big.js` 处理高精度小数
3. 格式化展示：`100000000` → `1亿`，提升可读性
4. 表单校验：限制用户输入位数，前端拦截超大值

```js
// BigInt 示例
const bigNum = 9007199254740991n + 1n;

// decimal.js 示例
const Decimal = require('decimal.js');
const result = new Decimal('0.1').plus('0.2'); // 精确 0.3
```

## 如何解决页面请求接口大规模并发问题

**背景**：数据分析平台需批量获取大量接口数据，不做并发控制会导致浏览器卡顿、请求失败。

**解决方案**：
1. **信号量/并发池**：控制同时请求数量
2. **滑动窗口**：时间窗口内限制请求数
3. **请求队列**：超出并发的请求排队等待

```js
class ConcurrencyPool {
  constructor(max = 3) {
    this.max = max;
    this.running = 0;
    this.queue = [];
  }

  async add(task) {
    return new Promise((resolve, reject) => {
      this.queue.push({ task, resolve, reject });
      this.next();
    });
  }

  async next() {
    if (this.running >= this.max || this.queue.length === 0) return;
    this.running++;
    const { task, resolve, reject } = this.queue.shift();
    try {
      const res = await task();
      resolve(res);
    } catch (e) {
      reject(e);
    } finally {
      this.running--;
      this.next();
    }
  }
}
```

## 大文件上传

**背景**：用户上传大文件（视频、压缩包等），直接上传容易超时失败，需要分片、断点续传、秒传能力。

**调研方案**：
1. 直接上传：文件大时容易超时，无进度反馈
2. 分片上传：将文件切分为多个 chunk 并行上传
3. 断点续传：记录已上传分片，中断后从断点继续
4. 秒传：通过文件 hash 判断服务器是否已有该文件

**方案落地**：

```js
class FileUploader {
  constructor(file, options = {}) {
    this.file = file;
    this.chunkSize = options.chunkSize || 2 * 1024 * 1024; // 2MB
    this.concurrent = options.concurrent || 3;
    this.uploadUrl = options.uploadUrl;
    this.chunks = this._splitChunks();
  }

  // 分片
  _splitChunks() {
    const chunks = [];
    for (let i = 0; i < this.file.size; i += this.chunkSize) {
      chunks.push(this.file.slice(i, i + this.chunkSize));
    }
    return chunks;
  }

  // 计算文件 hash（用于秒传）
  async _calcHash() {
    // 使用 spark-md5 计算文件 hash
    const spark = new SparkMD5.ArrayBufferBuilder();
    const buffer = await this.file.slice(0, 1024 * 1024).arrayBuffer(); // 取前1MB
    spark.append(buffer);
    return spark.end();
  }

  // 秒传检测
  async checkExists() {
    const hash = await this._calcHash();
    const res = await fetch(`${this.uploadUrl}/check`, {
      method: 'POST',
      body: JSON.stringify({ hash, size: this.file.size })
    });
    return res.json();
  }

  // 分片上传
  async upload() {
    const existRes = await this.checkExists();
    if (existRes.exists) {
      console.log('秒传成功');
      return existRes.fileUrl;
    }

    // 查询已上传分片（断点续传）
    const uploaded = await this._getUploadedChunks();

    const pool = new ConcurrencyPool(this.concurrent);
    const promises = this.chunks.map((chunk, index) => {
      if (uploaded.includes(index)) return Promise.resolve();
      return pool.add(() => this._uploadChunk(chunk, index));
    });

    await Promise.all(promises);
    return this._mergeChunks();
  }

  async _uploadChunk(chunk, index) {
    const formData = new FormData();
    formData.append('chunk', chunk);
    formData.append('index', index);
    formData.append('total', this.chunks.length);
    formData.append('filename', this.file.name);
    return fetch(`${this.uploadUrl}/chunk`, { method: 'POST', body: formData });
  }

  async _mergeChunks() {
    return fetch(`${this.uploadUrl}/merge`, {
      method: 'POST',
      body: JSON.stringify({ filename: this.file.name })
    });
  }
}
```

**反思**：
- 分片大小需要权衡：太小增加请求数，太大降低并行效果
- 秒传用前 1MB 计算 hash 即可，全量计算太慢
- 需要考虑网络中断后的本地状态持久化（IndexedDB）

## 前端权限控制

**背景**：企业后台系统需要精细的权限控制，包括谁能访问哪些页面、能看到哪些按钮、能操作哪些数据。

**RBAC 模型**：用户 → 角色 → 权限。用户分配角色，角色绑定权限。

```
用户 A → 角色[管理员] → 权限[用户管理, 数据导出, 系统配置]
用户 B → 角色[普通用户] → 权限[数据查看]
```

**路由级权限**：

```js
// 动态路由注册
router.beforeEach((to, from, next) => {
  const roles = store.state.user.roles;
  const hasPermission = to.meta.roles?.some(role => roles.includes(role));
  if (hasPermission || !to.meta.roles) {
    next();
  } else {
    next('/403');
  }
});
```

**按钮级权限**：

```js
// 自定义指令 v-permission
Vue.directive('permission', {
  inserted(el, binding) {
    const required = binding.value;
    const userPermissions = store.state.user.permissions;
    if (!userPermissions.includes(required)) {
      el.parentNode?.removeChild(el);
    }
  }
});

// 使用
<!-- <button v-permission="user:add">新增用户</button> -->
```

**数据级权限**：
- 后端在 SQL 查询时根据用户角色/部门过滤数据
- 前端通过接口参数传递权限标识，后端返回对应数据范围

**反思**：
- 权限数据缓存到 localStorage，减少请求
- 权限变更时需要刷新路由和状态
- 注意权限校验的边界情况（如未登录、token 过期）

## 前端安全

**XSS（跨站脚本攻击）**：
- 原理：恶意脚本注入到页面中执行
- 防御：输入过滤、输出转义、CSP 策略、HttpOnly Cookie

```js
// 转义 HTML
function escapeHtml(str) {
  const map = { '&': '&amp;', '<': '&lt;', '>': '&gt;', '"': '&quot;', "'": '&#x27;' };
  return str.replace(/[&<>"']/g, c => map[c]);
}
```

**CSRF（跨站请求伪造）**：
- 原理：利用用户已登录的 Cookie，伪造请求
- 防御：SameSite Cookie、CSRF Token、验证 Referer

**点击劫持**：
- 原理：透明 iframe 覆盖在页面上，诱导用户点击
- 防御：`X-Frame-Options: DENY`、`Content-Security-Policy: frame-ancestors 'none'`

**DNS 劫持**：
- 原理：DNS 解析被篡改，指向恶意域名
- 防御：HTTPS（证书校验）、DNS over HTTPS

**子域名劫持**：
- 原理：子域名未配置，攻击者注册该子域名的 CNAME
- 防御：所有子域名配置空页面或 CNAME 到主域

## 首屏优化：从 5s 到 1s

**背景**：首屏加载过慢导致用户流失，需要从 5s 优化到 1s 以内。

**调研方案**：
1. SSR（服务端渲染）：服务端生成 HTML，首屏快但增加服务器压力
2. 预渲染（Prerender）：构建时生成静态 HTML，适合内容不常变的页面
3. 骨架屏：先展示骨架占位，提升感知速度
4. 资源优化：压缩、Tree Shaking、Code Splitting、CDN
5. 预加载：`preload`、`prefetch`、`preconnect`

**方案落地**：

```html
<!-- 关键资源预加载 -->
<link rel="preload" href="/app.js" as="script">
<link rel="preconnect" href="https://api.example.com">
```

```js
// Code Splitting - 路由懒加载
const Dashboard = () => import(/* webpackChunkName: "dashboard" */ './Dashboard.vue');

// Tree Shaking - 按需导入
import { Button } from 'element-plus'; // 而非 import * as ElementPlus
```

**优化清单**：
- 开启 Gzip/Brotli 压缩
- 图片使用 WebP 格式，懒加载
- 第三方库 CDN 引入
- 移除未使用的 CSS（PurgeCSS）
- HTTP/2 多路复用
- 关键 CSS 内联，非关键 CSS 异步加载

**效果**：5s → 2s（资源优化）→ 1.2s（SSR/预渲染）→ 1s（骨架屏提升感知）

## WebSocket 长连接

**背景**：实时通讯场景（聊天、直播弹幕、协作编辑），需要稳定的长连接。

**心跳机制**：

```js
class WebSocketManager {
  constructor(url) {
    this.url = url;
    this.ws = null;
    this.heartbeatInterval = 30000;
    this.reconnectDelay = 1000;
    this.maxReconnectAttempts = 5;
    this.reconnectAttempts = 0;
    this.messageQueue = [];
  }

  connect() {
    this.ws = new WebSocket(this.url);

    this.ws.onopen = () => {
      this.reconnectAttempts = 0;
      this._startHeartbeat();
      this._flushQueue();
    };

    this.ws.onmessage = (e) => {
      const data = JSON.parse(e.data);
      if (data.type === 'pong') return;
      this._onMessage(data);
    };

    this.ws.onclose = () => {
      this._stopHeartbeat();
      this._reconnect();
    };

    this.ws.onerror = () => {
      this.ws.close();
    };
  }

  send(data) {
    if (this.ws?.readyState === WebSocket.OPEN) {
      this.ws.send(JSON.stringify(data));
    } else {
      this.messageQueue.push(data);
    }
  }

  _startHeartbeat() {
    this.heartbeatTimer = setInterval(() => {
      this.send({ type: 'ping' });
    }, this.heartbeatInterval);
  }

  _stopHeartbeat() {
    clearInterval(this.heartbeatTimer);
  }

  _reconnect() {
    if (this.reconnectAttempts >= this.maxReconnectAttempts) return;
    this.reconnectAttempts++;
    setTimeout(() => this.connect(), this.reconnectDelay * this.reconnectAttempts);
  }

  _flushQueue() {
    this.messageQueue.forEach(msg => this.send(msg));
    this.messageQueue = [];
  }
}
```

**关键设计**：
- 指数退避重连，避免频繁请求
- 消息队列保证离线消息不丢失
- 心跳检测连接存活状态
- 状态同步：连接恢复后拉取增量数据

## 在线白板/画板

**背景**：需要实现支持多人协作的在线白板，包含图形绘制、撤销重做、实时同步。

**Canvas vs SVG**：

| 维度 | Canvas | SVG |
|------|--------|-----|
| 性能 | 大量元素性能好 | 元素少时性能好 |
| 交互 | 需要手动计算命中 | DOM 事件直接绑定 |
| 缩放 | 需要重绘 | 矢量无损缩放 |
| 适用场景 | 大量图形、游戏 | 少量图形、图表 |

**撤销重做**：

```js
class UndoRedo {
  constructor() {
    this.history = [];
    this.index = -1;
  }

  push(action) {
    this.history = this.history.slice(0, this.index + 1);
    this.history.push(action);
    this.index++;
  }

  undo() {
    if (this.index > 0) {
      this.index--;
      return this.history[this.index].undo();
    }
  }

  redo() {
    if (this.index < this.history.length - 1) {
      this.index++;
      return this.history[this.index].redo();
    }
  }
}
```

**CRDT 协同**：
- 使用 Yjs 管理白板状态
- 每个图形作为文档中的一个元素
- 通过 WebSocket 广播状态变更

**背景**：产品需要支持多语言，用户可动态切换语言。

**i18n 方案**：

```js
// 消息文件
const messages = {
  zh: { greeting: '你好', welcome: '欢迎使用' },
  en: { greeting: 'Hello', welcome: 'Welcome' }
};

// i18n 实例
const i18n = {
  locale: 'zh',
  messages,

  t(key) {
    const keys = key.split('.');
    let val = this.messages[this.locale];
    for (const k of keys) {
      val = val?.[k];
    }
    return val || key;
  },

  setLocale(locale) {
    this.locale = locale;
    document.documentElement.lang = locale;
  }
};

// 使用
console.log(i18n.t('greeting')); // 你好
```

**动态切换**：
- 语言包按需加载：`import(`./locales/${locale}.json`)`
- 切换语言时更新所有文本节点
- 持久化用户语言偏好到 localStorage

```js
option = {
```

## 前端打包体积优化：从 2MB 到 300KB

**背景**：首屏 JS 包体积过大，影响加载速度。

**优化手段**：

```js
// webpack.config.js
module.exports = {
  optimization: {
    splitChunks: {
      chunks: 'all',
      cacheGroups: {
        vendor: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendors',
          priority: 10
        }
      }
    }
  },
  externals: {
    vue: 'Vue',
    'element-plus': 'ElementPlus' // CDN 引入
  }
};
```

**优化清单**：
1. Tree Shaking：确保 sideEffects 配置正确，按需引入
2. Code Splitting：路由懒加载，按路由分割
3. 第三方库 CDN 化：Vue、Lodash 等通过 CDN 引入
4. 图片压缩：tinypng，使用 WebP
5. Gzip/Brotli 压缩：服务端开启
6. 移除 console.log：Terser 压缩时去除
7. 分析包体积：webpack-bundle-analyzer 定位大依赖
8. 替换大依赖：lodash → lodash-es / loadash 按需引入

**效果**：2MB → 800KB（Tree Shaking + CDN）→ 500KB（Code Splitting）→ 300KB（Gzip 后）

## 前端 CI/CD

**背景**：需要自动化构建、测试、部署流程，提高发布效率和安全性。

**流程设计**：

```yaml
# .github/workflows/deploy.yml
name: CI/CD
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: 18
      - run: npm ci
      - run: npm run lint
      - run: npm run test
      - run: npm run build
      - uses: actions/upload-artifact@v3
        with:
          name: dist
          path: dist/

  preview:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/download-artifact@v3
        with:
          name: dist
      - run: npx serve -s . -p 3000 &
      # 部署到预览环境（Vercel / Netlify）

  deploy:
    needs: preview
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    steps:
      - run: |
          # 部署到生产环境
          scp -r dist/ user@server:/var/www/app
```

**灰度发布**：
- Nginx 按 Cookie/权重分流到新版本
- 功能开关（Feature Flag）控制新特性灰度
- 监控灰度用户指标，异常时快速回滚

**预览环境**：
- 每个 PR 自动部署到独立预览 URL
- 评论通知预览链接
- 合并后自动清理预览环境

```js
function reverseList(head) {
  let prev = null;
  let curr = head;
  while (curr) {
    const next = curr.next;
    curr.next = prev;
    prev = curr;
    curr = next;
  }
  return prev;
}

// 递归法
function reverseListRecursive(head) {
  if (!head || !head.next) return head;
  const newHead = reverseListRecursive(head.next);
  head.next.next = head;
  head.next = null;
  return newHead;
}
```

### 链表环检测

```js
// 快慢指针（Floyd 判圈算法）
function hasCycle(head) {
  let slow = head;
  let fast = head;
  while (fast && fast.next) {
    slow = slow.next;
    fast = fast.next.next;
    if (slow === fast) return true;
  }
  return false;
}

// 找环入口
function detectCycle(head) {
  let slow = head;
  let fast = head;
  while (fast && fast.next) {
    slow = slow.next;
    fast = fast.next.next;
    if (slow === fast) break;
  }
  if (!fast || !fast.next) return null;
  let ptr = head;
  while (ptr !== slow) {
    ptr = ptr.next;
    slow = slow.next;
  }
  return ptr;
}
```

### 二叉树遍历

```js
class TreeNode {
  constructor(val) {
    this.val = val;
    this.left = null;
    this.right = null;
  }
}

// 前序遍历
function preOrder(root) {
  if (!root) return [];
  return [root.val, ...preOrder(root.left), ...preOrder(root.right)];
}

// 中序遍历
function inOrder(root) {
  if (!root) return [];
  return [...inOrder(root.left), root.val, ...inOrder(root.right)];
}

// 后序遍历
function postOrder(root) {
  if (!root) return [];
  return [...postOrder(root.left), ...postOrder(root.right), root.val];
}

// 层序遍历
function levelOrder(root) {
  if (!root) return [];
  const result = [];
  const queue = [root];
  while (queue.length) {
    const size = queue.length;
    const level = [];
    for (let i = 0; i < size; i++) {
      const node = queue.shift();
      level.push(node.val);
      if (node.left) queue.push(node.left);
      if (node.right) queue.push(node.right);
    }
    result.push(level);
  }
  return result;
}
```

### 排序算法

```js
// 快速排序
function quickSort(arr) {
  if (arr.length <= 1) return arr;
  const pivot = arr[Math.floor(arr.length / 2)];
  const left = arr.filter(x => x < pivot);
  const middle = arr.filter(x => x === pivot);
  const right = arr.filter(x => x > pivot);
  return [...quickSort(left), ...middle, ...quickSort(right)];
}

// 归并排序
function mergeSort(arr) {
  if (arr.length <= 1) return arr;
  const mid = Math.floor(arr.length / 2);
  const left = mergeSort(arr.slice(0, mid));
  const right = mergeSort(arr.slice(mid));
  return merge(left, right);
}

function merge(left, right) {
  const result = [];
  let i = 0, j = 0;
  while (i < left.length && j < right.length) {
    if (left[i] <= right[j]) result.push(left[i++]);
    else result.push(right[j++]);
  }
  return result.concat(left.slice(i)).concat(right.slice(j));
}
```

### LRU 缓存

```js
class LRUCache {
  constructor(capacity) {
    this.capacity = capacity;
    this.cache = new Map();
  }

  get(key) {
    if (!this.cache.has(key)) return -1;
    const val = this.cache.get(key);
    this.cache.delete(key);
    this.cache.set(key, val);
    return val;
  }

  put(key, value) {
    if (this.cache.has(key)) {
      this.cache.delete(key);
    } else if (this.cache.size >= this.capacity) {
      this.cache.delete(this.cache.keys().next().value);
    }
    this.cache.set(key, value);
  }
}
```

```js
function debounce(fn, delay = 300) {
  let timer = null;
  return function (...args) {
    clearTimeout(timer);
    timer = setTimeout(() => {
```

### 节流

```js
function throttle(fn, delay = 300) {
  let lastTime = 0;
  return function (...args) {
    const now = Date.now();
    if (now - lastTime >= delay) {
      fn.apply(this, args);
      lastTime = now;
    }
  };
}
```

### Promise 相关实现

```js
// 手写 Promise
class MyPromise {
  constructor(executor) {
    this.state = 'pending';
    this.value = undefined;
    this.reason = undefined;
    this.onFulfilledCallbacks = [];
    this.onRejectedCallbacks = [];

    const resolve = (value) => {
      if (this.state === 'pending') {
        this.state = 'fulfilled';
        this.value = value;
        this.onFulfilledCallbacks.forEach(cb => cb());
      }
    };

    const reject = (reason) => {
      if (this.state === 'pending') {
        this.state = 'rejected';
        this.reason = reason;
        this.onRejectedCallbacks.forEach(cb => cb());
      }
    };

    try {
      executor(resolve, reject);
    } catch (e) {
      reject(e);
    }
  }

  then(onFulfilled, onRejected) {
    return new MyPromise((resolve, reject) => {
      const fulfilled = () => {
        try {
          const result = onFulfilled(this.value);
          result instanceof MyPromise ? result.then(resolve, reject) : resolve(result);
        } catch (e) {
          reject(e);
        }
      };

      const rejected = () => {
        try {
          const result = onRejected(this.reason);
          result instanceof MyPromise ? result.then(resolve, reject) : resolve(result);
        } catch (e) {
          reject(e);
        }
      };

      if (this.state === 'fulfilled') fulfilled();
      else if (this.state === 'rejected') rejected();
      else {
        this.onFulfilledCallbacks.push(fulfilled);
        this.onRejectedCallbacks.push(rejected);
      }
    });
  }

  catch(onRejected) {
    return this.then(null, onRejected);
  }

  static resolve(value) {
    return new MyPromise(resolve => resolve(value));
  }

  static reject(reason) {
    return new MyPromise((_, reject) => reject(reason));
  }

  static all(promises) {
    return new MyPromise((resolve, reject) => {
      const results = [];
      let count = 0;
      promises.forEach((p, i) => {
        MyPromise.resolve(p).then(val => {
          results[i] = val;
          count++;
          if (count === promises.length) resolve(results);
        }, reject);
      });
    });
  }

  static race(promises) {
    return new MyPromise((resolve, reject) => {
      promises.forEach(p => {
        MyPromise.resolve(p).then(resolve, reject);
      });
    });
  }
}

// Promise.all 手写
function promiseAll(promises) {
  return new Promise((resolve, reject) => {
    const results = [];
    let completed = 0;
    promises.forEach((p, index) => {
      Promise.resolve(p).then(
        value => {
          results[index] = value;
          completed++;
          if (completed === promises.length) resolve(results);
        },
        reject
      );
    });
  });
}

// Promise.retry 重试
function promiseRetry(fn, retries = 3, delay = 1000) {
  return new Promise((resolve, reject) => {
    function attempt(n) {
      fn().then(resolve, (err) => {
        if (n <= 0) return reject(err);
        setTimeout(() => attempt(n - 1), delay);
      });
    }
    attempt(retries);
  });
}
```
