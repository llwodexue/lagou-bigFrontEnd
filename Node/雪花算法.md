## 雪花算法

雪花算法（Snowflake Algorithm）是由 Twitter 开发的一种分布式 ID 生成算法，旨在生成全局唯一且有序的 64 位整数 ID。这种算法特别适合在高并发、分布式系统中生成唯一 ID

> 雪花算法原生设计是为了生成 64 位的 ID，但如果需要生成 32 位的 ID，可以通过对原算法进行一些调整来实现

**算法原理**

雪花算法的核心思想是利用时间戳、机器标识和序列号的组合来保证 ID 的全局唯一性和有序性：

- **时间戳**：记录 ID 生成时刻，保证 ID 整体递增趋势
- **机器标识**：区分不同机器或进程，避免多节点生成重复 ID
- **序列号**：同一毫秒内同一机器可能生成多个 ID，通过自增序列号来区分

这种设计使得：
1. 生成的 ID 全局唯一（时间戳 + 机器标识 + 序列号组合）
2. ID 大致按时间递增（有利于数据库索引性能）
3. 完全本地化生成，不依赖数据库或外部服务，性能极高

**雪花算法的 ID 结构**

```bash
# 64位结构
1 bit (符号位) | 41 bits (时间戳) | 10 bits (机器标识) | 12 bits (序列号)
# 32位结构
1 bit (符号位) | 10 bits (时间戳) | 5  bits (机器标识) | 16 bits (序列号)
```

- 符号位（1 bit）：恒为 0，表示 ID 为正数
- 时间戳（41 bit）：精确到毫秒，可容纳约 69 年
- 机器标识（10 bit）：前 5 位为数据中心 ID，后 5 位为机器 ID，最多支持 1024 个节点
- 序列号（12 bit）：同一毫秒内可生成 4096 个 ID

**JavaScript 实现**

```js
class Snowflake {
  constructor(workerId = 1, datacenterId = 1, epoch = 1577836800000) {
    this.workerId = workerId
    this.datacenterId = datacenterId
    this.epoch = epoch // 自定义起始时间戳
    this.sequence = 0
    this.lastTimestamp = -1
  }

  nextId() {
    let timestamp = Date.now()

    if (timestamp === this.lastTimestamp) {
      this.sequence = (this.sequence + 1) & 4095 // 12 bit 序列号
      if (this.sequence === 0) {
        // 序列号溢出，等待下一毫秒
        timestamp = this.waitNextMillis(this.lastTimestamp)
      }
    } else {
      this.sequence = 0
    }

    if (timestamp < this.lastTimestamp) {
      throw new Error('时钟回拨，拒绝生成 ID')
    }

    this.lastTimestamp = timestamp

    const delta = timestamp - this.epoch
    // 注意：JavaScript 的 Number.MAX_SAFE_INTEGER 为 2^53 - 1，
    // 而雪花算法生成的是 64 位 ID，超出安全整数范围可能导致精度丢失。
    // 生产环境建议使用字符串或 BigInt 存储雪花 ID。
    return (delta << 22) | (this.datacenterId << 17) | (this.workerId << 12) | this.sequence
  }

  waitNextMillis(lastTimestamp) {
    let timestamp = Date.now()
    while (timestamp <= lastTimestamp) {
      timestamp = Date.now()
    }
    return timestamp
  }
}

const sf = new Snowflake(1, 1)
console.log(sf.nextId())
```

**注意事项**

JavaScript 中 `Number.MAX_SAFE_INTEGER` 为 `2^53 - 1`（9007199254740991），而雪花算法生成的 64 位 ID 最大值为 `2^63 - 1`。当 ID 超过安全整数范围时，位运算会出现精度丢失。生产环境中建议：
- 使用字符串格式存储和传输雪花 ID
- 或使用 `BigInt` 类型来处理大整数
- 或使用缩短版雪花算法（如 53 位版本）来适配 JavaScript 环境
